## 三、发现的问题与 Harness 更新

### 问题一：ESLint 不认识测试文件的全局变量

```
现象：
  pnpm check 报大量 'vi' is not defined、'describe' is not defined 等错误

根因：
  vite.config.ts 里配置了 globals: true（Vitest 运行时注入）
  tsconfig.json 里配置了 types: ["vitest/globals"]（TypeScript 认识）
  但 ESLint 不知道这些全局变量存在

修复：
  在 eslint.config.js 里给测试文件加 globals 配置：
  files: ['src/**/*.test.{ts,tsx}', 'src/**/*.spec.{ts,tsx}', 'src/test/**/*.ts']
  globals: { vi, describe, it, test, expect, beforeEach, afterEach, ... }

Harness 更新：eslint.config.js 新增测试文件全局变量声明
```

### 问题二：setup.ts 的 process 变量不被 ESLint 识别

```
现象：
  src/test/setup.ts 里的 process.env 报 'process' is not defined

根因：
  process 是 Node.js 全局变量，浏览器环境的 ESLint 配置不认识

修复：
  在测试文件的 globals 配置里加上 process: 'readonly'

Harness 更新：eslint.config.js 测试文件 globals 加入 process
```

### 问题三：require-yield 规则误报

```
现象：
  测试里的空 async generator（模拟空响应）触发 ESLint require-yield 错误

根因：
  ESLint 要求 generator 函数至少有一个 yield

修复：
  空 generator 里加 yield ''

经验：
  这不是 Harness 的问题，是测试写法的问题
  但说明 Agent 在写测试时需要注意 ESLint 规则
```

### 问题四：request.test.ts 的 mock 方式需要调整

```
现象：
  第一版测试的 axios mock 方式不对
  interceptors.request.use 被 mock 成空函数
  导致拦截器回调没有被捕获，6 个测试失败

根因：
  mock 时没有保存拦截器注册的回调函数

修复：
  重写 mock，在 interceptors.request.use 的 mock 实现里
  把 fulfilled 和 rejected 回调保存到外部变量
  测试时直接调用这些回调来验证拦截器逻辑

经验：
  测试 axios 拦截器需要特殊的 mock 技巧
  不能简单地 mock 成空函数
```
