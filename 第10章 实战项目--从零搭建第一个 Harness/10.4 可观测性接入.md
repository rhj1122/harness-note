# 10.4　可观测性接入

> 本节记录 Vitest 测试配置和第一批测试文件的编写过程。

---

## 一、Vitest 配置

### 已有配置（vite.config.ts）

```typescript
test: {
  environment: 'jsdom',       // 模拟浏览器环境
  globals: true,              // 全局注入 describe/it/expect/vi
  setupFiles: ['./src/test/setup.ts'],
  passWithNoTests: true,      // 无测试文件时不报错
  coverage: {
    provider: 'v8',
    thresholds: { lines: 70 },
  },
}
```

### tsconfig.json 补充

```json
"types": ["vitest/globals", "node"]
```

让 TypeScript 认识 `vi`、`describe`、`expect` 等全局变量，以及 `process.env`。

### setup.ts 全局配置

```typescript
import '@testing-library/jest-dom'

// 环境变量控制 console 输出
if (process.env.VITEST_SILENT === 'true') {
  vi.spyOn(console, 'log').mockImplementation(() => {})
  vi.spyOn(console, 'error').mockImplementation(() => {})
  vi.spyOn(console, 'warn').mockImplementation(() => {})
}
```

### 命令设计

```json
"test:run": "cross-env VITEST_SILENT=true vitest run",     // 默认屏蔽 console
"test:run-with-log": "vitest run",                          // 需要看日志时用
```

## 二、编写的测试文件

### chatStore.test.ts（Store 测试）

测试 Zustand Store 的四个方法：
- `addMessage` — 添加消息
- `updateLastMessage` — 更新最后一条消息
- `clearMessages` — 清空消息
- `setLoading` — 修改 loading 状态

**踩过的坑：** 解构出来的原始类型值（`isLoading`）不会随 Store 更新而变化，必须重新 `getState()` 读取。

### LoginPage.test.tsx / ChatPage.test.tsx（组件测试）

测试页面骨架是否正确渲染。

**学到的查询方法：**
- `screen.getByText()` — 按文本内容查找
- `screen.getByPlaceholderText()` — 按 placeholder 查找
- `screen.getByRole()` — 按语义角色查找（推荐）

### useAIChat.test.ts（Hook 测试）

测试 Hook 的完整行为：
- 初始状态验证
- `sendMessage` 添加用户消息和 AI 占位消息
- `sendMessage` 调用 `streamMessage` API
- 空内容不发送
- API 失败时显示错误消息
- `clearMessages` 清空消息

**关键技术：**
- `renderHook()` — 在测试环境运行 Hook
- `act()` — 包裹状态更新操作
- `vi.mock()` — 替换 API 模块，不发真请求
- `vi.mocked().mockImplementation()` — 控制 mock 函数行为

## 五、经验总结

```
测试的优先级：
  Hooks > Store > Components/Pages
  业务逻辑最重要，UI 渲染其次

Mock 的核心思路：
  用假函数替代真的 API 调用
  控制假函数返回什么 → 验证 Hook 的行为是否正确

常见坑：
  Hook 不能在 beforeEach 里直接调用（必须用 renderHook）
  解构出来的原始类型值不会自动更新（要重新 getState）
  ESLint v10 + Vitest globals 需要在 tsconfig 里加 types
  Windows 环境设置环境变量需要 cross-env
```
