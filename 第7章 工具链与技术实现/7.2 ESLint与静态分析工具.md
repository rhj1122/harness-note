# 7.2　ESLint 与静态分析工具

> 本节目标：掌握前端 Harness 中 ESLint 的完整配置方案，以及其他有用的静态分析工具的使用方法。

---

## 一、ESLint 配置架构

Harness 的 ESLint 配置分为三层：

```
第一层：基础规则（社区标准）
  eslint:recommended
  @typescript-eslint/recommended
  → 覆盖语法错误、类型安全基础

第二层：框架规则（React 相关）
  eslint-plugin-react
  eslint-plugin-react-hooks
  → 覆盖 React 最佳实践、Hooks 规则

第三层：自定义 Harness 规则（架构约束）
  eslint-rules/no-api-in-components.js
  eslint-rules/no-ai-sdk-in-components.js
  eslint-rules/no-store-in-pages.js
  → 覆盖项目特定的架构边界
```

---

## 二、完整的 ESLint 配置文件

```javascript
// eslint.config.js（ESLint v9 flat config）
import js from '@eslint/js'
import tsPlugin from '@typescript-eslint/eslint-plugin'
import tsParser from '@typescript-eslint/parser'
import reactPlugin from 'eslint-plugin-react'
import reactHooksPlugin from 'eslint-plugin-react-hooks'

// 导入自定义 Harness 规则
import noApiInComponents from './eslint-rules/no-api-in-components.js'
import noAiSdkInComponents from './eslint-rules/no-ai-sdk-in-components.js'
import noStoreInPages from './eslint-rules/no-store-in-pages.js'

export default [
  // 第一层：基础规则
  js.configs.recommended,
  {
    files: ['src/**/*.{ts,tsx}'],
    languageOptions: {
      parser: tsParser,
      parserOptions: { project: './tsconfig.json' },
    },
    plugins: { '@typescript-eslint': tsPlugin },
    rules: {
      ...tsPlugin.configs.recommended.rules,
      '@typescript-eslint/no-explicit-any': 'warn',      // any 使用警告
      '@typescript-eslint/no-unused-vars': 'error',      // 未使用变量报错
    },
  },

  // 第二层：React 规则
  {
    files: ['src/**/*.{ts,tsx}'],
    plugins: {
      react: reactPlugin,
      'react-hooks': reactHooksPlugin,
    },
    rules: {
      ...reactPlugin.configs.recommended.rules,
      ...reactHooksPlugin.configs.recommended.rules,
      'react-hooks/exhaustive-deps': 'warn',
    },
    settings: { react: { version: 'detect' } },
  },

  // 第三层：自定义 Harness 规则
  {
    files: ['src/**/*.{ts,tsx}'],
    plugins: {
      harness: {
        rules: {
          'no-api-in-components': noApiInComponents,
          'no-ai-sdk-in-components': noAiSdkInComponents,
          'no-store-in-pages': noStoreInPages,
        },
      },
    },
    rules: {
      'harness/no-api-in-components': 'error',
      'harness/no-ai-sdk-in-components': 'error',
      'harness/no-store-in-pages': 'error',
    },
  },

  // 忽略文件
  {
    ignores: ['dist/', 'node_modules/', '*.config.js'],
  },
]
```

## 三、自定义规则的文件组织

```
项目根目录/
├── eslint-rules/
│   ├── README.md              ← 规则说明索引
│   ├── no-api-in-components.js
│   ├── no-ai-sdk-in-components.js
│   ├── no-store-in-pages.js
│   └── no-direct-fetch.js     ← 禁止直接使用 fetch
└── eslint.config.js
```

`eslint-rules/README.md` 的内容：

```markdown
# 自定义 ESLint 规则

## 架构层级规则（ARCH）
| 规则 | 级别 | 说明 |
|------|------|------|
| no-api-in-components | error | Components 层不能直接导入 API 层 |
| no-store-in-pages | error | Pages 层不能直接导入 Store 层 |

## 编码规范规则（CONV）
| 规则 | 级别 | 说明 |
|------|------|------|
| no-ai-sdk-in-components | error | Components 层不能直接使用 AI SDK |
| no-direct-fetch | error | 禁止在 src/components/ 和 src/pages/ 中直接使用 fetch |
```

## 四、常用的补充静态分析工具

### TypeScript 严格模式增强

```json
// tsconfig.json 推荐配置
{
  "compilerOptions": {
    "strict": true,
    "noUnusedLocals": true,
    "noUnusedParameters": true,
    "noImplicitReturns": true,
    "exactOptionalPropertyTypes": true,  // 更严格的可选属性检查
    "noUncheckedIndexedAccess": true     // 数组访问返回 T | undefined
  }
}
```

### madge：循环依赖检测

```bash
# 安装
npm install -D madge

# 检测循环依赖
npx madge --circular src

# 生成依赖图（可视化）
npx madge --image dependency-graph.svg src

# 只检测特定目录
npx madge --circular src/hooks
```

在 CI 中使用：
```yaml
- name: 循环依赖检测
  run: |
    result=$(npx madge --circular src)
    if [ -n "$result" ]; then
      echo "发现循环依赖："
      echo "$result"
      exit 1
    fi
```

### ts-prune：死代码检测

```bash
# 安装
npm install -D ts-prune

# 检测未使用的导出
npx ts-prune

# 输出示例：
# src/hooks/useOldFeature.ts:1 - useOldFeature (unused)
# src/utils/deprecated.ts:5 - formatOldDate (unused)
```

### Prettier：代码格式化

```json
// .prettierrc
{
  "semi": false,
  "singleQuote": true,
  "tabWidth": 2,
  "trailingComma": "es5",
  "printWidth": 100
}
```

```json
// package.json
{
  "scripts": {
    "format": "prettier --write src",
    "format:check": "prettier --check src"
  }
}
```

## 五、把所有工具整合进 npm scripts

```json
// package.json
{
  "scripts": {
    "dev": "vite",
    "build": "vite build",

    // 单独运行各工具
    "type-check": "tsc --noEmit",
    "lint": "eslint src --ext .ts,.tsx",
    "lint:fix": "eslint src --ext .ts,.tsx --fix",
    "format:check": "prettier --check src",
    "format": "prettier --write src",
    "circular": "madge --circular src",
    "dead-code": "ts-prune",

    // 组合命令（Agent 常用）
    "check": "npm run type-check && npm run lint",
    "check:all": "npm run format:check && npm run type-check && npm run lint && npm run circular",

    // 测试
    "test": "vitest",
    "test:run": "vitest run",
    "test:coverage": "vitest run --coverage"
  }
}
```

Agent 的标准验证流程：
```bash
npm run check && npm run test:run
```

## 六、在 AGENTS.md 中声明工具

```markdown
## 静态分析工具
- `npm run check`：类型检查 + ESLint（每次修改后必须运行）
- `npm run test:run`：运行测试（修改逻辑后必须运行）
- `npm run circular`：检测循环依赖（修改 import 结构后运行）
- `npm run dead-code`：检测未使用的导出（清理任务时运行）

自定义 ESLint 规则说明：eslint-rules/README.md
```

## 本节小结

```
ESLint 三层配置：
  基础规则（社区标准）
  框架规则（React + Hooks）
  自定义 Harness 规则（架构约束）

自定义规则组织：
  eslint-rules/ 目录
  每条规则一个文件
  README.md 作为规则索引

补充静态分析工具：
  madge：循环依赖检测
  ts-prune：死代码检测
  Prettier：代码格式化

npm scripts 整合：
  npm run check（type-check + lint）
  npm run check:all（全量检查）
  npm run test:run（单次测试）
```
