# 7.5　可观测性工具栈

> 本节目标：了解前端项目可观测性工具的选择，掌握轻量级本地方案的配置，以及关键监控指标的仪表盘设计。

---

## 一、前端可观测性的范围

前端的可观测性和后端不同，分为两个维度：

```
维度一：开发时可观测性（Agent 用）
  目的：让 Agent 能验证代码是否正确
  工具：tsc、ESLint、Vitest、Playwright
  → 第五章已经详细讲过

维度二：运行时可观测性（用户行为 + 性能）
  目的：了解真实用户的使用情况和性能表现
  工具：错误监控、性能监控、用户行为分析
  → 本节重点
```

运行时可观测性对 Harness Engineering 的价值：

```
Agent 修复了一个 bug
  → 测试通过，构建成功
  → 但真实用户环境下还有问题吗？

运行时监控告诉你：
  - 线上错误率是否下降了？
  - 页面加载时间是否改善了？
  - 用户是否还在遇到同样的问题？
```

### 错误监控：Sentry（免费版）

```bash
npm install @sentry/react @sentry/vite-plugin
```

```typescript
// src/main.tsx
import * as Sentry from '@sentry/react'

Sentry.init({
  dsn: import.meta.env.VITE_SENTRY_DSN,
  environment: import.meta.env.MODE,
  // 只在生产环境启用
  enabled: import.meta.env.PROD,
  // 采样率（开发时设为 0，生产时设为 0.1-1.0）
  tracesSampleRate: 0.1,
})
```

```typescript
// src/providers/ErrorProvider.tsx
import * as Sentry from '@sentry/react'

// 捕获未处理的错误
export const ErrorBoundary = Sentry.ErrorBoundary

// 手动上报错误
export function reportError(error: Error, context?: Record<string, unknown>) {
  Sentry.captureException(error, { extra: context })
}
```

在 AGENTS.md 里声明：
```markdown
## 错误处理
- 所有未捕获的错误通过 ErrorBoundary 上报到 Sentry
- 手动上报使用 src/providers/ErrorProvider.tsx 中的 reportError()
- 禁止在组件里直接调用 Sentry.captureException()
```

### 性能监控：Web Vitals

```bash
npm install web-vitals
```

```typescript
// src/utils/reportWebVitals.ts
import { onCLS, onFID, onFCP, onLCP, onTTFB } from 'web-vitals'

export function reportWebVitals() {
  // 上报到 Sentry 或自定义后端
  const report = (metric: { name: string; value: number }) => {
    console.log(`[Web Vitals] ${metric.name}: ${metric.value}`)
    // 或者上报到监控平台
  }

  onCLS(report)   // Cumulative Layout Shift
  onFID(report)   // First Input Delay
  onFCP(report)   // First Contentful Paint
  onLCP(report)   // Largest Contentful Paint
  onTTFB(report)  // Time to First Byte
}
```

### 用户行为埋点：自定义 TrackingProvider

```typescript
// src/providers/TrackingProvider.tsx
type TrackEvent = {
  name: string
  properties?: Record<string, unknown>
}

function track(event: TrackEvent) {
  // 开发环境只打印，生产环境上报
  if (import.meta.env.DEV) {
    console.log('[Track]', event.name, event.properties)
    return
  }
  // 上报到埋点平台（神策、GrowingIO 等）
  window.sensors?.track(event.name, event.properties)
}

export function useTracking() {
  return { track }
}
```

## 四、AI 调用的专项监控

AI H5 应用有一类特殊的监控需求：**AI 调用质量监控**。

```typescript
// src/providers/AIProvider.tsx（添加监控）
export function useAIProvider() {
  const { track } = useTracking()

  async function* streamMessage(messages: Message[]) {
    const startTime = Date.now()
    let tokenCount = 0
    let hasError = false

    try {
      const stream = await aiClient.chat.completions.create({
        model: appConfig.aiModel,
        messages,
        stream: true,
      })

      for await (const chunk of stream) {
        const content = chunk.choices[0]?.delta?.content || ''
        tokenCount += content.length / 4  // 粗略估算 token 数
        yield content
      }
    } catch (error) {
      hasError = true
      throw error
    } finally {
      // 上报 AI 调用指标
      track({
        name: 'ai_stream_complete',
        properties: {
          duration_ms: Date.now() - startTime,
          estimated_tokens: Math.round(tokenCount),
          has_error: hasError,
          model: appConfig.aiModel,
        },
      })
    }
  }

  return { streamMessage }
}
```

关键 AI 监控指标：
```
首 Token 延迟（TTFT）：从发送请求到收到第一个 token 的时间
流式完成时间：整个流式响应的总时长
Token 消耗：每次对话的 token 数量（影响成本）
错误率：AI 调用失败的比例
用户中断率：用户主动停止流式响应的比例
```

## 六、在 Harness 中集成可观测性

可观测性工具需要在 Harness 中有对应的约束：

```markdown
# AGENTS.md 补充（可观测性相关）

## 可观测性规范
- 错误上报：使用 src/providers/ErrorProvider.tsx 中的 reportError()
- 埋点：使用 src/providers/TrackingProvider.tsx 中的 useTracking()
- AI 监控：AIProvider 已内置，不需要额外处理
- 禁止：直接调用 Sentry、sensors 等 SDK（必须通过 Provider）
```

对应的 ESLint 规则：

```javascript
// eslint-rules/no-direct-sentry.js
// 禁止在 Provider 层以外直接调用 Sentry
module.exports = {
  create(context) {
    return {
      ImportDeclaration(node) {
        const filePath = context.getFilename()
        const importPath = node.source.value

        const isProvider = filePath.includes('/providers/')
        const importsSentry = importPath.startsWith('@sentry/')

        if (!isProvider && importsSentry) {
          context.report({
            node,
            message:
              '[CONV-005] 禁止直接导入 Sentry SDK。\n' +
              '原因：错误上报必须通过 src/providers/ErrorProvider.tsx 统一管理。\n' +
              '修复：使用 import { reportError } from "@/providers/ErrorProvider"',
          })
        }
      },
    }
  },
}
```

## 本节小结

```
前端可观测性两个维度：
  开发时（Agent 用）：tsc + ESLint + Vitest + Playwright
  运行时（用户行为）：错误监控 + 性能监控 + 埋点

轻量级方案（个人/小团队）：
  错误监控：Sentry 免费版
  性能监控：Web Vitals
  埋点：自定义 TrackingProvider

AI 应用专项监控：
  首 Token 延迟、流式完成时间、Token 消耗、错误率
  在 AIProvider 里统一埋点

Harness 集成：
  所有监控 SDK 通过 Provider 封装
  ESLint 规则禁止直接调用 SDK
  在 AGENTS.md 里声明规范
```
