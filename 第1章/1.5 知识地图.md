## 一、三大支柱 × 五大实践领域

Harness Engineering 的知识体系可以用一张矩阵来理解：

```
                    ┌──────────────────────────────────────────┐
                    │           Harness Engineering            │
                    │         核心等式：Agent = Model + Harness │
                    └──────────────────────────────────────────┘
                                        │
              ┌─────────────────────────┼─────────────────────────┐
              ▼                         ▼                         ▼
    ┌─────────────────┐      ┌─────────────────┐      ┌─────────────────┐
    │  上下文工程      │      │   架构约束       │      │   熵管理         │
    │ Context Eng.    │      │ Arch. Constraints│      │ Entropy Mgmt.   │
    │                 │      │                 │      │                 │
    │ 让 Agent 看到   │      │ 机械阻止 Agent   │      │ 持续对抗代码     │
    │ 正确的信息       │      │ 犯结构性错误     │      │ 腐化             │
    └────────┬────────┘      └────────┬────────┘      └────────┬────────┘
             │                        │                        │
             ▼                        ▼                        ▼
    AGENTS.md / docs /       分层架构 / ESLint /        黄金原则 /
    skills / 工具发现         CI 流水线 / 边界            清理 Agent /
                                                        质量度量
```

**五大实践领域**贯穿三大支柱：

| 实践领域 | 涉及章节 | 核心问题 |
|---------|---------|---------|
| 文档工程 | 第二章 | 如何让 Agent 获取正确信息？ |
| 架构设计 | 第三章 | 如何用机械约束保护代码结构？ |
| 质量治理 | 第四章 | 如何持续对抗 AI Slop？ |
| 可观测性 | 第五章 | 如何让 Agent 感知环境、验证结果？ |
| 工作流设计 | 第六章 | 如何把以上一切组合成高效的工作流？ |

## 二、各章依赖关系与学习路径

```
第一章：认识 Harness Engineering（你现在在这里）
    │
    │  建立概念基础，没有这章后面都看不懂
    ▼
第二章：上下文工程
    │
    │  学会写 AGENTS.md 和 docs/ 体系
    │  这是 Harness 的"大脑"——Agent 靠它理解项目
    ▼
第三章：架构约束
    │
    │  学会设计前端分层架构和 ESLint 规则
    │  这是 Harness 的"骨架"——机械阻止错误
    │  依赖第二章：架构规则需要在文档中描述
    ▼
第四章：熵管理
    │
    │  学会设计黄金原则和清理流程
    │  这是 Harness 的"免疫系统"——持续对抗腐化
    │  依赖第二、三章：黄金原则是文档，清理依赖架构约束
    ▼
第五章：可观测性与反馈回路
    │
    │  学会搭建让 Agent 能"看到"结果的基础设施
    │  这是 Harness 的"神经系统"——感知与反馈
    │  依赖前四章：反馈回路需要文档、约束、质量标准
    ▼
第六章：实践模式与工作流设计
    │
    │  把前五章整合成完整的工作流
    │  这是 Harness 的"操作手册"
    ▼
第七章：工具链与技术实现
    │
    │  具体工具的选型和配置
    │  可以在学习前六章时同步参考
    ▼
第八章：典型案例分析
    │
    │  真实案例，巩固和深化理解
    ▼
第九章：思维模式升级
    │
    │  从"写代码者"到"系统设计者"的思维转变
    ▼
第十章：实战项目
    └── 从零搭建你的第一个完整 Harness（综合实践）
```

## 三、学习路径建议

### 路径 A：系统学习（推荐）

按章节顺序，第一章 → 第十章，每章学完做对应的小练习。

适合：想彻底掌握这门学科，未来在工作中独当一面。

### 路径 B：快速上手

```
第一章（概念基础）
    ↓
第二章 2.1-2.2（AGENTS.md 核心）
    ↓
第三章 3.1-3.3（架构约束核心）
    ↓
第六章 6.1（从零构建第一个 Harness）
    ↓
回头补完其余章节
```

适合：有紧急项目需求，需要快速为前端项目搭建一个可用的 Harness。

### 路径 C：工具导向

```
第七章（工具链）→ 第二章 → 第三章 → 第五章
```

适合：已有一定 AI 编码经验，想快速了解具体工具。
